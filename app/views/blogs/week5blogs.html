What are classes?


Because Ruby is a Object Oriented Programming Language, we use classes to define objects and methods.Lets start the example with an example shall we.
class Mustang
end

this is an example of an empty class. I named it mustang because I have a mustang (car) and it is what popped into my head as an example. Inside of this class Mustang we can write the codes for Mustang. This way when we call Mustang as a class it will run the code inside of it. So lets say we code in that this class Mustang has code inside that is equal to four seats, 5 gears, steering wheel, brakes and a 530 hp engine (plus all the other car stuff). And when we call the class Mustang the code runs and we get a 530hp 4 seater mustang.


Now if we were to do things with it, like add a driver or owner, we could just callin the class and avoid re-writing the whole car again. We can also leave the Mustang with out an engine for example and when we call it, we can add paint and an engine. For example:
black_v8_530hp = Mustang.new

And then we add the info we need.

silver_v6_305hp = Mustang.new

And now we change it because 530hp is too fast.


This is basically how classes work, with out getting into the dirty nasty scary code words. But if you wanted to read more click here. This has good information about classes that I used often to learn more about them.
